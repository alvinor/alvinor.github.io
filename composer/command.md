用法:

>  命令 [选项] [参数]

 

 
```
Options:

  -h, --help                     显示帮助信息

  -q, --quiet                    静默方式

  -V, --version                  版本信息

      --ansi                     强制 ANSI 输出

      --no-ansi                 禁止ANSI 输出

  -n, --no-interaction          不提示交互信息

      --profile                  显示时间和内存使用情况

      --no-plugins               是否禁用插件ns.

  -d, --working-dir=WORKING-DIR 如果定义则作为工作目录

  -v|vv|vvv, --verbose           扩展输出: 1 正常, 2 详细 3 调试

 

有效命令

  about           显示关于composer的相关信息

  archive         存档.当前composer包信息

  browse          在浏览器中打开镜像的URL或者首页.

  clear-cache     清除包的换成缓存

  clearcache      清除包的换成缓存

  config            设置配置项

  create-project  由给定的包创建一个项目到指定目录

  depends         显示已经安装包的依赖关系

  diagnose        诊断系统识别一般错误

  dump-autoload   Dumps the autoloader.

  dumpautoload    Dumps the autoloader.

  exec            执行一个容器里面的二进制或者脚本

  global          全局运行($COMPOSER_HOME).

  help            帮助

  home            在浏览器中打开镜像的URL或者首页.

  info            显示包的信息

  init           在当前目录初始化composer.json

  install       从composer.lock 安装依赖，如果没有则从composer.json安装

  licenses       依赖证书嘻嘻你

  list            列出命令

  outdated        显示包的更新信息

  prohibits       显示哪些程序包阻止给定的包安装。.

  remove        从require或者  require-dev移除一个包.

  require         给composer.json 增加依赖包并安装

  run-script     运行 composer.json中定义的脚本

  search          查询包

  self-update     更新composer到最新包.

  selfupdate      更新composer到最新包.

  show            显示包的信息packages.

  status         显示被修改的本地包.

  suggests       显示包的建议

  update         根据composer.json更新本地包，并修改composer.lock文件

  validate        验证composer.json 和 composer.lock.

  why             显示已经安装包的安装成功的依赖包

 

  why-not        显示组织安装包的不能安装成功的依赖包
  
  ```
